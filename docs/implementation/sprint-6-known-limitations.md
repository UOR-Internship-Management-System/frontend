# Sprint 6 Known Limitations

1. Critical Playwright execution uses deterministic OpenAPI-shaped route mocks. End-to-end conformance against a deployed backend, authentication service, persistence layer, and real CSV processing infrastructure remains a release-environment responsibility.
2. The implementation is pull-request ready, not declared production ready, until backend conformance and deployment-environment validation pass.
3. The Student deep-dive route is an approved detail shell; Sprint 6's complete official-record inspection workflow is delivered within Academic Ledger as read-only UI.
4. No edit, delete, rollback, internship-management, candidate-filtering, or shortlist workflows were added because they are outside the approved reduced scope.
5. Visual fidelity was implemented from the supplied HTML references and verified through responsive browser behavior; automated pixel-diff baselines are not part of this repository.

There are no known frontend contract blockers, failing required checks, or uncommitted required changes at closure.
